● 两套系统的核心区别如下：

  1. 分数处理方式不同

  ┌──────────┬─────────────────────────┬──────────────────────────────┐
  │   方面   │ 普通版本 (default.yaml) │   有界版本 (bounded.yaml)    │
  ├──────────┼─────────────────────────┼──────────────────────────────┤
  │ 分数范围 │ 原始 GRS [6, 30]        │ 归一化到 [0, 1]              │
  ├──────────┼─────────────────────────┼──────────────────────────────┤
  │ 归一化   │ 无归一化                │ 训练前归一化：(grs - 6) / 24 │
  ├──────────┼─────────────────────────┼──────────────────────────────┤
  │ 模型输出 │ 无界输出                │ Sigmoid 确保输出在 [0, 1]    │
  └──────────┴─────────────────────────┴──────────────────────────────┘

  2. 模型架构差异

  普通版本 (surgical_qa_model.py):
  - 使用 FusionRegressor - 普通多层感知机
  - 最后一层没有激活函数，输出无界

  有界版本 (surgical_qa_model_bounded.py):
  - 使用 BoundedFusionRegressor (带 Sigmoid)
  - 最后一层用 Sigmoid，确保输出 ∈ [0, 1]
  - 提供 denormalize_score() 反归一化函数

  3. 数据加载器差异

  普通版本 (data_loader.py):
  SurgicalQADataLoader -> SurgicalVideoDataset
  # 直接返回原始分数 score

  有界版本 (data_loader_normalized.py):
  SurgicalQADataLoaderNormalized -> SurgicalVideoDatasetNormalized
  # 返回归一化分数 score ∈ [0, 1]
  # 保留 score_original 字段

  4. 推理方式差异

  有界版本推理时需要反归一化：
  score_normalized = model(video, mask)  # [0, 1]
  score_original = model.denormalize_score(score_normalized)  # [6, 30]

  5. 为什么需要两套系统？

  有界版本的优势：
  - 训练稳定性：Sigmoid 输出梯度更稳定，无爆炸问题
  - 收敛更快：归一化后数值范围统一
  - 符合先验：已知分数范围，用 Sigmoid 强制约束

  普通版本适用于：
  - 分数范围未知或变化较大
  - 不希望强约束输出范围

  简而言之：有界版本 = 归一化标签 + Sigmoid 回归器 + 反归一化推理

  ┌──────────────┬──────────────┬──────────────────┐

  - 有界版本：inference_bounded.py 支持反归一化到不同范围

  ---
  🔴 有界版本的严重逻辑错误

  错误 1：Epoch 循环错误（致命）

  位置：train_bounded.py:288
  # ❌ 错误
  for epoch in range(start(start_epoch), config['epochs']):

  问题：start() 是 Python 内置函数，会将 start_epoch
  当作可迭代对象并返回第一个元素。这会导致：
  - 如果 start_epoch=0 → 抛出 TypeError: 'int' object is not iterable
  - 如果 start_epoch 是列表/字符串 → 返回第一个元素，完全错误

  正确写法：
  # ✅ 正确
  for epoch in range(start_epoch, config['epochs']):

  ---
  错误 2：Mask 加载拼写错误

  位置：train_bounded.py:422 (evaluate 函数中)
  # ❌ 错误：检查 'm' 而不是 'masks'
  if 'm' in batch and batch['masks'] is not None:

  问题：条件永远不会为真，因为 batch 的键是 'masks' 而不是 'm'。导致 masks
  始终为 None，模型无法获取掩膜监督信号。

  正确写法：
  # ✅ 正确
  if 'masks' in batch and batch['masks'] is not None:

  错误 3：数据加载器初始化参数不一致

  位置：train_bounded.py:500-514

  dataloaders = SurgicalQADataLoaderNormalized(
      data_root=config['data_root'],
      batch_size=config['batch_size'],
      num_workers=config['num_workers'],
      dataset_kwargs={

          # ❌ 缺少从配置文件读取归一化参数！
          'score_min': 6.0,   # 硬编码

  问题：
  1. 参数名错误：'score' 应该是 'score_max'
  2. 硬编码而非从 config 读取，与 bounded.yaml 配置不一致
  3. 如果配置文件修改了这些值，不会被使用

  正确写法：

      # ...

      'score_max': config.get('score_max', 30.0),  # 修正参数名
      'target_min': config.get('target_min', 0.0),
      'target_max': config.get('target_max', 1.0)
  }

  错误 4：denormalize_score 默认参数与存储参数不匹配

  位置：surgical_qa_model_bounded.py:188-230

  def denormalize_score(self, score_normalized, target_min=None, 
  target_max=None,
                      norm_min=None, norm_max=None):
      # ...

  默认 1.0

  问题：
  - 归一化时：[6, 30] → [0, 1]（data_loader_normalized.py）
  - 模型初始化存储：self.score_min=1.0,

  ❓ 为什么会出现这么多错误？

  你的分析是对的！从代码特征来看，有界版本是在普通版本的基础上复制粘贴修改的，但
  修改不彻底：

  1. 复制粘贴引入的语法错误：range(start(start_epoch), ...) 显然是误触了 Python
  内置函数
  2. 拼写/手误没有修正：'m' in batch 这种低级错误
  3. 硬编码而非使用配置：明明有 config 变量却硬编码数值
  4. 分数范围混乱：初始按 [1,30] 编写，后续改到 [6,30] 时没有完全更新所有地方
  5. 缺少系统性验证：修改后没有统一测试，导致配置文件与代码默认值不一致

  这反映了**"复制-修改但审查不彻底"**的典型问题，特别是在涉及多个文件的参数传递
  时容易出现不一致。
